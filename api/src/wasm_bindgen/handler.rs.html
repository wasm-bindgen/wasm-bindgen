1//! Hooks for responding to hard-abort and reinit events on the Wasm instance.
2//!
3//! # Hard abort
4//!
5//! A hard abort occurs when the WebAssembly instance encounters a
6//! non-recoverable error — an `unreachable` instruction, out-of-memory, or
7//! stack overflow — that cannot be caught by Rust's `catch_unwind`.  The
8//! instance is poisoned and no further exports can be called.
9//!
10//! Use [`set_on_abort`] to register a callback that runs at the moment of
11//! termination.  Returns the previously registered handler (`None` if unset),
12//! mirroring the `std::panic::set_hook` convention.
13//!
14//! **Only available when built with `panic=unwind`.**
15//! [`set_on_abort`] returns `None` and the callback will never fire on
16//! `panic=abort` builds. Support for `panic=abort` may be added in a future
17//! release.
18//!
19//! # Reinit
20//!
21//! [`schedule_reinit()`] signals that the instance should be reinitialized.
22//! The next call to any export detects this, creates a fresh
23//! `WebAssembly.Instance` from the same module.
24//!
25//! Works with both `panic=unwind` and `panic=abort` builds.
26//!
27//! The reinit machinery is automatically emitted when [`schedule_reinit()`] is
28//! used — no CLI flag is required. `--experimental-reset-state-function` is
29//! only needed for the public `__wbg_reset_state()` export.
30#[doc(hidden)]
31pub use crate::__rt::schedule_reinit;
32#[doc(hidden)]
33pub use crate::__rt::set_on_abort;
